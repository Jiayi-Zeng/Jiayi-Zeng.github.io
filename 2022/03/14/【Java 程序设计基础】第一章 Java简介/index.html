Java 简介

March 14, 2022 3338

Java特点，执行系统，程序设计环境，第一个Java程序，Java程序规范，应用程序和小应用程序

书本：Java面向对象程序设计（第2版 ）袁绍欣，等

Java特点

Simple 简单

并不需要：头文件、结构体、指针、共同体、运算符重载、虚基类

Object-Oriented 面向对象的

与C++的不同之处在于：用更简单的接口代替了多重继承

Architecture-Neutral 高效的

  1. write once, run anywhere.
  2. 编译器生成一个与体系结构无关的对象文件格式：字节代码；
  3. 作为系统一部分的库定义了可移植的接口。
  4. 指定了原始数据类型的大小，其中string类型被保存在一个标准的Unicode格式中。

    Portable 可移植的

    High-Performance 结构中立的

    c/c++: 源代码 → 机器代码
    Java: 源代码 → 字节码 → 机器代码

    Robust 鲁棒性

  5. later dynamic (runtime) checking；
  6. Java has a pointer model that eliminates the possibility of overwriting memory and corrupting data；
  7. Garbage Collection；

    Secure 安全的

  8. 在 Java 中执行一个程序时层层把关，而且活动范围被局限在特定空间。
  9. Java字节码 → 类装载器 → 字节码校验器 → 安全管理器

    Multithreaded 多线程的

  10. 多线程的好处是更好的交互响应和实时行为。
  11. 线程是一种程序将自己分割(称为“分裂”)为两个或多个同时(或伪同时)运行的任务的方法。
  12. Java是第一种支持并发编程的主流语言。

    Distributed 分布式的

  13. Java有大量用于处理TCP/IP协议(如HTTP和FTP)的扩展库。
  14. Java应用程序可以通过URLs通过网络打开和访问对象，就像访问本地文件系统一样容易。

    Dynamic 动态的

  15. 它的设计是为了适应不断变化的环境。
  16. 库可以自由地添加新的方法和实例变量，而不会对它们的客户端产生任何影响。

    Java 执行系统

    编写、编译和执行Java

  17. 编译器将.java代码转换为称为字节码的中间代码(Abc.class)。
  18. 字节码由类装入器装入到JVM中。
  19. 字节码验证器(JVM内建的程序)检查字节码的完整性，如果没有发现问题，就将其传递给解释器。
  20. JVM中的解释器将每一行字节码转换成可执行的机器码，并将其传递给操作系统/硬件，即CPU来执行。

    Java 虚拟机

    Java虚拟机(JVM)是一种虚拟机，它使计算机能够运行Java程序以及用其他语言编写的也被编译为Java字节码的程序。

    垃圾回收机制

  21. 垃圾收集器销毁无用的对象。
  22. 垃圾收集器是守护线程的最好例子，因为它总是在后台运行。
  23. 垃圾收集器的主要目标是通过销毁无法访问的对象来释放堆内存。

    Java程序设计环境

    JDK

    Java Development ToolKit(Java开发工具包)。
    JDK是整个JAVA的核心，包括了Java运行时环境(Java Runtime Envirnment) ， 一 些 Java 工 具 ( 例如：javac/java/jdb等)和Java基础的类库。

    JRE

    Java Runtime Enviromental(java运行时环境)。也就是我们说的JAVA平台，所有的Java程序都要在JRE下才能运行。包括JVM和JAVA核心类库和支持文件。与JDK相比，它不包含开发工具——编译器、调试器和其它工具。

    JVM

    Java Virtual Mechinal(JAVA虚拟机)。JVM是JRE的一部分，它是一个虚构出来的计算机，是通过在实际的计算机上仿真模拟各种计算机功能来实现的。JVM有自己完善的硬件架构，如处理器、堆栈、寄存器等，还具有相应的指令系统。JVM 的主要工作是解释自己的指令集(即字节码)并映射到本地的 CPU 的指令集或 OS 的系统调用。

    环境变量classpath的作用

    在执行java程序的时候，类是由JVM动态加载的(只有在某个类被首次用到的时候，才会加载这个类的bytecode)。

classpath用于告知JVM，到哪里去找这些类。

JVM按照以下顺序寻找类:

  1. bootstrap类: 这些类是Java平台的基础类；
  2. 扩展类: 位于JRE或存放扩展类的目录中的包(package)— jre/lib/ext/；
  3. 用户自定义的包

    HelloWorld.java

    Java程序规范

  4. 文件命名
    源文件扩展名为 .java
    源文件如果包含有一个public类，则必须按该类名命名（大小写相同)
  5. 类的个数
    一个源文件最多有一个public类，其他类的个数不限。
  6. main方法
    是Java Application执行入口，且有固定的格式：
  7. Java严格区分大小写
  8. Java方法由一条条语句构成，每条语句以分号结束
  9. 注释
  10. 标识符命名
    包名：全小写，一般为名词。
    类名：首字母大写，后面每一单词的首字母也大写。
    接口名：同类名。
    方法名：一般为动词，小写字母开头，每单词首字母大写。
    变量名：小写字母开头，一般为名词。后面每一单词的首字母大写
    常量名：基本类型的常量名全大写；而对象常量则为大小写混合。
